# Практическое задание: Deadlock и Race Condition

## Описание тестов

### testRaceConditionFixed
Проверяет исправление состояния гонки. `SafeCounter` должен использовать правильную синхронизацию для предотвращения состояния гонки.

### testDeadlockPrevention
Проверяет предотвращение дедлока через одинаковый порядок захвата ресурсов. Оба потока должны захватывать замки в одинаковом порядке.

### testSafeCounter
Проверяет потокобезопасный счетчик без состояния гонки. `SafeCounter` должен корректно работать в многопоточной среде.

### testReentrantLockTimeout
Проверяет использование `ReentrantLock` с тайм-аутом для предотвращения дедлока. Потоки должны использовать `tryLock(timeout)` для избежания бесконечного ожидания.

### testAtomicOperations
Проверяет использование атомарных операций для предотвращения состояния гонки. Все операции должны быть атомарными.

## Классы для реализации

- **SafeCounter**: Потокобезопасный счетчик без состояния гонки
